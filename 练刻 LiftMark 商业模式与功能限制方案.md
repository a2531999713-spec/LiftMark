# 练刻 LiftMark 完整商业方案

版本：v1.5  
方案类型：商业模式、会员权益、免费版限制、小组 Pro、激活码、后台发放、付费购买、运营规则  
不包含：技术架构、数据库设计、接口设计、代码实现

---

## 1. 商业模式总览

练刻 LiftMark 采用：

```text
未登录预览 + 登录免费版 + Pro 周期会员 + 永久会员 + 小组 Pro 权益
```

完整商业模式由以下部分组成：

1. 未登录预览模式。
    
2. 登录免费版。
    
3. Pro 周期会员。
    
4. 永久会员。
    
5. Pro 小组共享权益。
    
6. 激活码兑换。
    
7. 后台发放会员。
    
8. 付费购买会员。
    
9. 后期团队版、教练版、计划市场预留。
    

核心商业逻辑：

```text
免费版让用户和 1 个搭子能真实练起来。
Pro 让用户和训练小组练得更完整、更省事、更长期。
永久会员吸引早期重度用户和支持者。
小组 Pro 支持学生、宿舍、同学、训练搭子拼单使用。
激活码、后台发放、付费购买都统一获得会员权益。
```

## 2. 产品商业定位

练刻 LiftMark 不是普通健身打卡 App，也不是简单训练记录本。

产品定位：

```text
多人力量训练计划执行器 + 训练记录工具 + 进阶建议系统 + 计划分享平台
```

主要用户：

1. 普通力量训练者。
    
2. 健身搭子。
    
3. 学生训练小组。
    
4. 宿舍或同学训练组合。
    
5. 2-4 人固定训练小队。
    
6. 进阶训练者。
    
7. 后期私教、小团队、训练工作室。
    

核心付费理由：

```text
不是为了“记录一下”，而是为了让多人训练更清楚、更省事、更容易长期坚持。
```

## 3. 商业设计原则

### 3.1 免费版必须真实可用

免费版不能只是展示版。

免费版必须支持：

```text
可以开始正式训练
可以保存训练记录
可以和 1 个搭子一起练
可以创建 1 个基础小组
可以创建 1 个可编辑自定义训练方案
可以保存基础训练历史
可以体验多人训练的核心价值
```

不建议把“搭子一起练”从免费版去掉。因为练刻的核心差异化就是多人力量训练。如果免费版只能单人使用，用户很难感受到产品价值，也不容易拉搭子一起传播。

---

### 3.2 Pro 是完整能力，不是基础能力

Pro 不应该让用户感觉“不付费就不能用”。

Pro 的价值应该是：

```text
更多人
更多小组
更多计划
更完整编辑器
更高级分析
更完整同步
更适合长期训练
```

也就是：

```text
免费版够用，Pro 更完整。
```

---

### 3.3 小组共享是核心卖点

练刻的产品场景本来就是多人训练，所以 Pro 不能只按“单人会员”理解。

最终规则：

```text
一个 Pro / 永久会员账号，可以激活 2 个 Pro 小组。
每个 Pro 小组最多 4 人。
小组成员可以在该小组内共享小组 Pro 功能。
```

这允许学生、宿舍、训练搭子拼单使用，利于传播。

---

### 3.4 会员来源统一

用户获得会员的方式可以不同：

```text
付费购买
激活码兑换
后台发放
内测赠送
校园推广
合作方赠送
客服补偿
```

但用户获得的权益应该统一。

也就是说：

```text
不管用户是买的、兑换的、后台发的，最终都表现为 Pro 或永久会员。
```

不要让用户感受到复杂的来源差异。

## 7. 永久会员

永久会员是 Pro 的一次性购买方式，不是单独功能包。

永久会员包含：

```text
一次性解锁个人 Pro 权益。
可激活 2 个 Pro 小组。
每个 Pro 小组最多 4 人。
享受当前和后续合理范围内的个人 Pro 功能。
无广告。
```

### 永久会员边界

永久会员必须明确边界。

推荐文案：

```text
永久会员一次性解锁个人 Pro 权益。云同步、备份等在线服务按合理使用范围提供。团队版、教练版、计划市场付费内容及超出合理使用范围的云资源不包含在永久会员内。
```

永久会员不承诺：

```text
无限云存储
无限小组
无限成员
团队版永久免费
教练版永久免费
计划市场付费内容永久免费
所有未来商业功能永久免费
第三方服务永久免费
```

## 9. Pro 小组权益

## 9.1 小组权益规则

一个 Pro / 永久会员账号可以激活：

```text
2 个 Pro 小组
```

每个 Pro 小组最多支持：

```text
4 人
```

也就是说，一个会员账号最多可以覆盖：

```text
2 个小组 × 每组 4 人 = 8 人的训练场景
```

但这 8 人只是在对应小组内享受小组 Pro 能力，不是全部获得个人 Pro。

---

## 9.2 A 开会员，B 加入小组

场景：

```text
A 是 Pro / 永久会员
A 创建一个 Pro 小组
B 是免费用户
B 加入 A 的 Pro 小组
```

B 在 A 的 Pro 小组内可以使用：

```text
4 人训练
小组共享计划
小组训练分析
小组内训练记录同步
本小组内个人训练分析
小组成员趋势
小组计划执行进度
```

B 不会获得：

```text
自己的个人 Pro
自己的 2 个 Pro 小组资格
更多个人计划数量
多个个人自定义训练方案
其他小组的高级分析
自己创建 4 人 Pro 小组的能力
```

一句话规则：

```text
A 的会员可以让 B 在 A 的 Pro 小组内使用小组 Pro 功能，但不会让 B 获得全局 Pro。
```

## 9.3 拼单推广价值

这个设计允许学生和训练搭子拼单。

例如：

```text
永久会员 99.9 元
4 人小组一起使用
平均每人约 25 元
```

如果一个会员激活 2 个小组：

```text
最多可覆盖 2 个训练圈子
适合宿舍、同学、固定搭子和不同训练时间段的小组
```

这是推广优势，不是漏洞。

推广文案可以写：

```text
和搭子一起练更划算

开通 Pro 后，可激活 2 个 Pro 小组，每组最多 4 人。适合宿舍、同学、训练搭子和固定小队一起使用。
```

---

## 9.4 小组数量限制

最终规则：

```text
个人 Pro / 永久会员：默认 2 个 Pro 小组。
第 3 个 Pro 小组：后期可作为活动奖励或单独权益。
4 个及以上小组：归入后期团队版或教练版。
```

不建议个人会员默认给 4 个 Pro 小组，否则后期团队版难以收费。

```text

合作健身房
熟人测试
```

```text

客服补偿

```

后台可发放：

```text
Pro 30 天
Pro 90 天
Pro 180 天
Pro 365 天
永久会员
额外 Pro 小组名额，后期可选
```

后台发放时需要填写：

```text
用户
权益类型
有效期
来源
备注
可激活 Pro 小组数量
每组最大人数
```

后台发放原则：


不建议：

```text

给所有测试用户永久会员。
让激活码长期公开流通。
```

## 15. 数据可见性和隐私规则

训练数据涉及个人表现、1RM、身体数据和训练状态，需要有边界。

默认规则：

|数据类型|默认可见范围|
|---|---|
|小组总训练次数|小组成员可见|
|小组总训练量|小组成员可见|
|小组计划完成度|小组成员可见|
|成员是否完成训练|小组成员可见|
|成员详细重量 / 次数 / RPE|本人可见|
|成员 1RM|默认本人可见|
|成员身体数据|默认本人可见|
|个人进阶建议|默认本人可见|
|小组汇总分析|小组成员可见|
|详细个人分析|本人可见，可授权共享|

第一版可以先采用简单规则：

```text
本人可以看自己的详细数据。
小组成员可以看小组汇总。
组长可以看成员是否训练、完成情况和基础汇总。
成员详细训练数据、1RM、身体数据默认不公开。
```

后期再增加：

```text
允许向组长共享详细数据。
允许向全组共享训练表现。
允许隐藏个人记录。
允许只共享完成状态。
```

## 16. 会员到期和降级规则

会员到期后：

```text
不删除训练数据。
不删除小组。
不删除成员。
不删除计划。
不清空历史分析数据。
```

只做功能降级。

降级后规则：

```text
Pro 小组降为免费小组。
超过 2 人的小组成员保留，但不能新增成员。
超过免费限制的计划保留，但不能继续新增。
高级分析隐藏或降级为基础分析。
完整云同步降级为基础同步或暂停。
完整计划编辑器不可用。
续费后恢复 Pro 功能。
```

到期提示文案：

```text
Pro 已到期

你的训练数据已保留。重新开通后，可继续使用 4 人小组、高级分析、完整计划编辑器和完整同步。
```
