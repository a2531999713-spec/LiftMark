# UI Design Rules

All UI/UX design work MUST use the installed skills:

1. **taste-skill** (`~/.codex/skills/taste-skill`) - Anti-slop frontend design, layout optimization
2. **impeccable** (`~/.codex/skills/impeccable`) - Production-grade UI design, audit, polish

## When to use skills

- Any UI redesign or layout changes
- Adding new pages or components
- Improving existing page layouts
- Fixing visual hierarchy or spacing issues
- Color scheme or typography adjustments

## Workflow

1. Before any UI change, read the relevant skill SKILL.md
2. Apply the skill's design principles
3. Follow the skill's anti-pattern rules
4. Use the skill's terminology and conventions
