# Auth Flow

更新时间：2026-06-29

## 启动流程

```text
App 启动
  -> 读取本地 SecureStore session
  -> 无 session：进入 /account/login
  -> 有 session：轻量请求 /auth/me
  -> 在线且校验成功：进入 /(tabs)
  -> 有 session 但网络不可用：进入 /(tabs) 本机模式
```

本机模式只显示本地默认内容，不主动拉取云端完整资料，不做全量同步或云端恢复。

## 登录 / 注册流程

```text
打开 /account/login
  -> 勾选用户协议和隐私政策
  -> 输入手机号
  -> POST /auth/send-code
  -> 输入短信验证码
  -> POST /auth/login-with-code
  -> SecureStore 保存 session
  -> 进入 /(tabs)/today
```

当前版本移动端使用手机号 + 短信验证码登录 / 注册一体入口。新手机号由后端验证码登录接口自动创建账号。密码登录、密码注册接口保留在 service 层兼容后端，但不作为当前移动端主流程。前端不展示第三方登录、继续浏览或预览模式。

## 失败处理

- 手机号为空：请输入手机号。
- 手机号格式错误：请输入正确的手机号。
- 验证码为空或长度不正确：请输入 4-6 位短信验证码。
- 未勾选协议：请先同意用户协议和隐私政策。
- 服务器不可达：服务器连接失败，请检查网络或稍后再试。
- 验证码错误或过期：验证码错误或已过期。
- 服务器不可用不会清空 SQLite，不会清空已有离线 session。
