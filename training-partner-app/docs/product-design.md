## 1. 产品定位

“练刻 / LiftMark”是一个支持多人同练、计划自定义、自动重量计算、训练记录、周期进阶和计划分享的力量训练 App。

第一版不是普通单人训练记录工具，也不是 Excel 表格搬运，而是安卓本地可用的多人训练执行 App。

## 2. 用户角色

| 角色 | 需求 |
|---|---|
| 普通健身者 | 按计划训练、记录数据、获得下次建议 |
| 健身搭子 | 2-4 人一起练，同动作不同重量，轮流记录 |
| 进阶训练者 | 使用增力/增肌周期，关注 1RM、RPE/RIR 和减量 |
| 私教/小团队 | 后续分配计划、查看完成情况 |
| 新手用户 | 套用模板，降低训练计划理解成本 |

## 3. 核心业务流程

- 首次使用：seed 系统方案和默认用户计划 -> 创建默认小组 -> 添加成员 -> 输入 1RM -> 进入今日训练。
- 今日训练：读取小组当前用户计划/周期/周数 -> 按星期生成动作 -> 按恢复状态过滤 -> 按成员计算建议重量。
- 训练执行：创建 session -> 生成动作记录和 set -> 多人按动作轮换记录 -> 即时保存 SQLite。
- 动作替换：选择替代动作 -> 保存替代后的 exercise_id，同时保留 replaced_from_exercise_id。
- 训练总结：完成 session -> 统计完成度 -> 生成 progression_suggestions -> 进入总结页。
- 进阶建议：增力看完成情况和 RPE，增肌看次数区间、RIR 和动作质量。

## 5. 页面结构

- 底部 Tab：探索、搭子、训练、计划、记录。
- 探索页：计划推荐、动作库、创建计划、导入计划和搭子引导。
- 搭子页：本地训练小组、成员、1RM、训练统计、邀请入口和二维码加入预留。
- 训练页：当前计划、周期、周数、今日主题、恢复状态、成员建议重量、动作优先级和开始训练按钮。
- 计划页：当前计划、我的计划、系统方案、创建计划、导入计划、导出计划和本周安排。
- 记录页：训练历史、PR、预估 1RM、趋势、疲劳提示和下次建议。
- 设置页：不作为主 Tab，从页面右上角或功能入口进入，承载基础设置、数据管理、计划导入导出入口、SQLite/seed 调试信息。
- 训练执行页：从训练页进入，不显示底部 Tab，使用沉浸式深色全屏页面，突出当前动作、多成员记录卡、重量/次数 stepper、RPE/RIR 快捷输入、完成本组和自动保存状态。
- 总结页：完成度、本次训练总量、成员表现、预估 1RM、本次最佳动作、训练时长、下次建议和恢复建议。

## 6. 权限规则

第一版为本地单设备应用，不做账号登录和云端权限。数据模型保留角色：

- owner：组长，管理小组和计划。
- member：参与训练和记录。
- coach：后续教练模式预留。
- guest：后续只读查看预留。

## 7. 状态机

### 训练 session 状态

```text
draft -> in_progress -> completed
draft -> cancelled
in_progress -> cancelled
```

### 计划周期状态

```text
strength -> deload -> hypertrophy -> deload -> strength
```

### 进阶建议类型

```text
increase | maintain | decrease | deload | add_reps | maintain_or_decrease
```

### 恢复建议

```text
normal | remove_c | only_a | reduce_weight | deload | rest
```

## 8. 边界条件

- 用户跳过 1RM 时仍可进入 App，建议重量显示未设置。
- 周五默认休息，可选补弱；不练周五不需要补回来。
- 状态一般删除 C 动作，状态差只保留 A 动作，状态很差建议休息或减量。
- 器械被占时替换动作，但原计划动作不变。
- 历史记录必须保存当时的计划快照。
- 历史推算只能作为训练建议，不能写成医疗、伤病或绝对结论。
- `.liftmark.json` / `.liftmark` 计划文件不能包含训练记录、成员 1RM 或个人参数。
- 系统方案不是用户计划；用户必须点击“使用此方案”后才生成自己的训练计划。
- 训练记录不能直接绑定系统方案，只能绑定用户计划或训练时计划快照。

## 9. 异常流程

- seed 失败：提示重试，不进入半初始化状态。
- SQLite 写入失败：训练执行页显示错误并保留当前输入状态。
- 缺少成员：今日训练提示添加成员。
- 缺少计划：提示从系统方案复制、创建计划或导入计划。
- 无替代动作：显示空状态并允许返回。
- 导入计划版本不兼容：显示中文提示，不覆盖本地计划。
- 危险设置操作：必须二次确认。

## 10. 业务限制

- 计划是数据，不是代码。
- 计划模板和个人参数分离。
- 计划和训练记录分离。
- 多人逻辑从第一版就保留。
- 本地 SQLite 优先。
- Domain 层不依赖 UI。
- 训练记录不能用 AsyncStorage 保存。
- Excel 训练计划只整理为后续 seed 数据设计说明，不硬编码进页面组件。
