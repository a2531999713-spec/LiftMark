# 练刻 LiftMark UI 设计规范

更新时间：2026-06-14

## 1. 品牌

- 中文名：练刻
- 英文名：LiftMark
- 完整品牌：练刻 LiftMark
- 定位：以单人或多人力量训练为核心的训练计划、搭子协作、训练记录与进阶建议平台。
- 文案方向：你的力量训练计划执行器；记录每次训练，刻下持续进步。

## 2. 设计关键词

力量训练、专业、干净、训练现场友好、数据清晰。

## 3. 色彩系统

| Token | 颜色 | 用途 |
|---|---|---|
| `brand` | `#FF5A4D` | 主按钮、激活 Tab、关键进度、训练动作强调 |
| `brandDark` | `#E5483D` | 按压态、危险主按钮 |
| `brandSoft` | `#FFF1F0` | PR 提示、浅色强调卡片 |
| `accent` | `#4C7CFF` | 数据、趋势、次级强调 |
| `success` | `#19C37D` | 完成、稳定推进、已保存 |
| `warning` | `#FFB020` | 接近 PR、疲劳提醒 |
| `danger` | `#E5484D` | 清空、删除、不可逆操作 |
| `background` | `#F2F4F7` | 页面背景 |
| `surface` | `#FFFFFF` | 卡片背景 |
| `dark` | `#0D1B2A` | 沉浸式训练页、图标深底 |
| `textStrong` | `#0D1B2A` | 标题和关键数据 |
| `textMuted` | `#6B7280` | 说明和辅助信息 |
| `border` | `#E7EAF0` | 卡片、输入框、分割线 |

## 4. 字号层级

| Token | 大小 | 用途 |
|---|---:|---|
| `display` | 32 | 总结页完成度、关键数字 |
| `headline` | 26 | 页面主标题 |
| `title` | 20 | 卡片标题、当前动作 |
| `subtitle` | 17 | 区块标题 |
| `body` | 15 | 正文 |
| `bodySmall` | 13 | 元信息、说明 |
| `caption` | 11 | 标签、单位、小提示 |

## 5. 间距系统

基础间距使用 4px 倍数：`xs: 4`、`sm: 8`、`md: 12`、`lg: 16`、`xl: 24`、`xxl: 32`。页面左右安全间距为 16px，卡片默认内边距为 16px。

## 6. 圆角系统

- `xs: 6`：小标签、优先级徽章。
- `sm: 8`：按钮、输入框、小卡片。
- `md: 12`：标准内容卡片。
- `lg: 16`：Hero、提示卡、训练执行成员卡。
- `pill: 999`：胶囊按钮、状态标签。

## 7. 阴影系统

标准卡片使用轻阴影和细边框；Hero 卡片使用中等阴影；沉浸式训练页使用深色层级、半透明边框和弱阴影。

## 8. 按钮规范

- 主按钮：珊瑚红填充、白色文字、最小高度 48px，用于开始训练、保存、完成本组。
- 次按钮：白底或浅灰底、细边框，用于导入、导出、切换、查看。
- 危险按钮：危险色填充或红色边框，必须配合二次确认。
- 训练现场按钮高度建议 52-56px，文字清楚、点击面积大。
- 普通功能页不允许用大面积输入框堆叠成后台表单；只有成员编辑、计划编辑、历史补录等真实输入流程可以使用表单结构。
- “复制方案”“切换计划”“导入计划”等选择流程必须使用 App 风格确认卡或底部弹层，不得用系统默认 Alert 承载复杂选项。
- 新增 Alert 只用于危险确认、系统错误或极轻量输入错误；普通成功提示、信息说明和选择流程使用 `AppModalSheet` / AppCard。
- 未实现功能不要做成显眼可点击按钮后再弹“开发中”；应收进说明卡、标签或二级入口，保证主要入口都可用。

## 9. 卡片规范

- 页面卡片白底、细边框、轻阴影。
- 卡片内容先显示最重要信息：标题、关键数字、状态，再显示说明。
- 统计卡可使用 2 列网格。
- 动作卡必须显示动作名、A/B/C 优先级、组次、强度和进入/操作入口。

## 10. Tab 规范

底部导航固定为：

```text
探索 / 搭子 / 训练 / 计划 / 记录
```

激活态使用品牌红，非激活态使用中性灰。设置页不作为主 Tab，可从设置入口或关于入口进入。

## 11. 状态标签规范

- 状态好：品牌浅底 + 品牌文字。
- 一般：中性灰浅底。
- 状态差：警告浅底。
- 休息：蓝色浅底。
- 已完成 / 已保存：成功色。
- 开发中：中性浅底 + 次文字。

## 12. 动作优先级 A/B/C 标签规范

- A：红色，主项或核心动作。
- B：橙色，重要辅助动作。
- C：绿色，补充动作。
- 标签使用小尺寸、强对比、稳定宽高。

## 13. 训练执行页规范

- 不显示底部 Tab，使用全屏沉浸式深色页面。
- 顶部显示训练中、动作进度、组数进度和进度条。
- 当前动作信息突出，包含动作名、优先级、组次、强度、休息建议。
- 多成员记录使用白色卡片，每个人独立重量、次数、完成情况。
- 重量和次数使用大号 stepper，并允许直接输入，重量可为小数，次数为整数。
- 训练执行页只保留重量、次数、完成/跳过和备注，不再展示旧强度快捷输入。
- 主按钮固定为醒目的“完成本组”。
- 辅助按钮：跳过、替换动作、下一个动作。
- 底部显示计时和“已自动保存”。

## 13.1 记录页规范

- 记录页默认显示当前成员个人数据，顶部文案必须包含“当前成员”。
- 顶部右侧只保留一个日历/筛选类入口；补录、编辑最近等操作放入内容区紧凑按钮。
- 使用个人概览卡 + 2x2 小统计 + 基础趋势 + 可点击日历 + 紧凑训练列表。
- 小组汇总未实现时显示“开发中”，不得复用个人数据。
- 设计稿：`docs/ui/record-page-redesign.png`。

## 13.2 设置页规范

- 设置页使用移动设置中心结构，不做大表单页面。
- 顶部为品牌/状态卡，下面使用分组列表项。
- 分组至少包含通用设置、训练设置、计划导出、关于/激活、开发与诊断、调试保护。
- 未实现能力显示“开发中”标签或说明卡，不在设置页常驻展示 JSON、数据库调试长文本或成员编辑表单。
- 设计稿：`docs/ui/settings-page-redesign.png`。

## 14. 空状态规范

空状态必须说明用户下一步，不只显示“暂无数据”。关键空状态要给可操作入口，例如添加成员、去探索计划、开始第一场训练。

## 15. 调试保护规范

重建测试数据、训练记录维护、重置计划等调试保护必须二次确认。第一阶段调试保护保留入口时，若尚未真正实现删除，必须明确提示“开发中”。

## 16. 资源引用规范

- 品牌图通过 `src/assets/brand/index.ts` 的 `liftmarkBrandAssets` 引用。
- 训练氛围图通过 `src/assets/images/index.ts` 的 `liftmarkImages` 引用。
- 当前 Expo 图标位于 `training-partner-app/assets/brand/`：桌面图标使用 `app-icon-1024.png` 深色实底，Android adaptive foreground 使用透明 `adaptive-icon-foreground-white.png`，背景色由 `app.json` 的 `#0D1B2A` 提供，splash 使用透明居中的 `splash-logo.png`。
- 页面内品牌图位于 `training-partner-app/src/assets/brand/`，与桌面图标和 splash 资源区分使用。
- 不在页面里散落 hex 颜色、字号、圆角，统一从 `src/theme/` 引用。
