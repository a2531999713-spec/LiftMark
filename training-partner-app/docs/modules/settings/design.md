# Settings 模块设计文档

更新时间：2026-06-24

## 2026-06-24 账号状态设计补充

- 未登录状态不进入我的页或主 Tab，直接进入手机号验证码登录 / 注册页。
- 已有本地 session 但服务器不可达时显示为“本机模式”，允许进入默认主界面，但不拉取云端完整资料。
- 登录免费版显示当前账号、会员状态和云同步开发中 / 可测试入口；免费版不能被设计成无法训练。
- Pro / 永久会员状态在账号区以标签或二级页展示，不放入顶部主卡主视觉。
- 登录拦截弹层主按钮为“登录 / 注册”，不提供关闭后浏览。
- 未登录我的页顶部主卡只作为兼容状态，不再作为正常使用路径。
- Pro 拦截弹层主按钮为“查看 Pro 权益”，次按钮为“暂不开通”。

## 2026-06-24 我的页设计调整

- 顶部主卡定位为“账号 + 训练身份”，未登录时显示登录引导，已登录时显示昵称、练刻 ID、当前成员、所在小组和训练角色。
- 云同步、会员与激活、账号安全作为“账号与服务”列表项，不作为顶部主视觉。
- 数据与隐私分组必须强调训练数据默认不公开；删除、清空、注销类入口必须二次确认。
- 开发者诊断隐藏到版本号 7 连点，不再普通展示 SQLite、seed、调试日志或清空测试数据。

## 2026-06-14 设置页重设计

- 设置页从大块表单卡片改为移动端设置中心：顶部品牌/状态卡 + 分组列表项。
- 顶部品牌区使用 `src/assets/brand/logo-primary.png` 等 LiftMark 品牌专属图，不使用普通健身 Hero 图替代。
- 分组包括：通用设置、训练设置、试用与激活、数据管理、计划管理、开发与诊断、危险操作。
- “成员资料”展示全部成员摘要并进入成员列表，不直接出现成员编辑表单、保存按钮或稍后补充按钮。
- “加重单位”展示按成员分别设置杠铃/哑铃加重单位的入口，进入全成员列表后由用户手动选择成员。
- “周五策略”不再作为全局 App 设置暴露；兼容字段仍留在数据库中。
- “最近导出预览”不再常驻展示，避免把设置页做成数据库调试页。
- 未实现能力保留“开发中”标签或说明卡，不把显眼入口做成点击后只弹系统 Alert。
- 设置页可通过“本地小组规则”入口说明当前本地小组与未来云同步区别，但不在主页面堆长段落。
- 导出完成后必须明确说明“当前版本暂未保存到文件”，并提供复制内容；设置页不常驻展示 JSON 预览。
- 导入计划成功后使用 App 风格弹层询问是否设为当前训练计划。

## 1. 设计目标

设置页作为本地优先 MVP 的安全控制面板：能让用户看到当前 App 和数据状态，并访问导出、导入、重置等高风险能力。

## 2. 核心业务规则

- App 品牌显示为“练刻 LiftMark”。
- 导出完成后显示 App 风格提示：内容已生成但当前版本暂未保存到文件，可复制内容；文件保存/分享作为后续增强，不在设置页常驻展示 JSON。
- `.liftmark` 计划文件只包含计划模板、动作库、替代动作和进阶规则，不包含成员 1RM 或训练记录。
- 导入计划必须生成新的本地 ID，避免覆盖既有计划。
- 导入计划落库后来源为用户计划，不修改系统方案，不把系统方案直接设为当前训练计划。
- 重置默认计划只补齐 seed 缺失数据，不删除训练记录。
- 清空测试数据暂不真正执行删除，等待软删除/清空策略确认。

## 3. 输入输出

- 输入：SQLite Repository、导出服务、计划文件服务、系统文件选择器。
- 输出：设置页状态、导出内容复制提示、导入成功确认弹层、开发中说明卡。

## 4. 边界条件

- SQLite 初始化失败时显示中文错误。
- 未实现功能应收进说明卡或开发中标签，不使用粗糙系统 Alert 承载普通信息。
- 危险操作必须二次确认。
