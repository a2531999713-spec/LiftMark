# Activation 模块概览

更新时间：2026-06-12

## 1. 模块职责

Activation 模块当前作为“会员与激活”二级入口的激活码兑换能力。第一版用户可见兑换以后端会员权益为准；旧本地试用状态仅作为历史兼容实现保留，不再作为普通用户入口展示。

## 2. 第一版能力

- 登录后调用后端 `/api/activation-codes/redeem` 兑换激活码。
- 会员状态由 `/api/membership/me` 返回。
- 不在普通用户页面显示本地试用模式、试用剩余天数或开发期测试码。
- 旧本地激活状态保留为兼容代码，不作为当前产品主状态。

## 3. 边界

- 激活码兑换需要登录账号。
- App 不保存阿里云 AccessKey，不直接调用短信或支付服务。
- 权益变化不影响本地训练数据完整性。
