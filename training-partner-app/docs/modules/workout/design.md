# Workout 模块设计文档

更新时间：2026-06-30

## 2026-06-30 训练执行增强

- RPE 是可选高级记录，默认折叠；展开后使用横向数字选择器，不能恢复 RIR。
- 休息状态必须展示倒计时、建议休息、已休息、下一组和下一位成员。
- 训练中替换动作只影响当前 session，保存 `replaced_from_exercise_id`，不修改原计划。
- 替换动作入口应是辅助操作，不抢重量、次数、完成本组的主路径。

## 2026-06-29 记录范围选择

- 开始训练前必须选择本次记录范围：仅我记录或小组成员。
- 选择小组成员时，只有被选中的成员会生成本次计划组；未参与成员不出现在训练执行页。
- 今日训练页把计划日选择和动作筛选放在开始训练之前，避免用户先进入训练再发现练错训练日或记录对象。
- 自由训练入口进入补录训练，避免创建没有动作快照的 session。

## 1. 设计目标

让 `workout` 模块拥有清晰边界，支撑多人训练 MVP，并为后续云同步、计划编辑器或商业版扩展保留空间。

## 2. 模块职责

- WorkoutSession、WorkoutExerciseRecord、WorkoutSet。
- 从今日训练生成 session、exercise records、workout sets。
- 训练执行页默认按动作轮换，当前动作下按成员记录 set。
- 每次修改立即保存 SQLite，离开页面后可恢复。

## 3. 非职责

- 不定义计划模板本身。
- 不直接决定进阶建议，只在完成训练后触发 progression。
- 不把训练记录保存到 AsyncStorage。

## 4. 核心业务规则

- 训练现场输入必须低摩擦，不能频繁要求键盘输入。
- 训练进行中退出 App 不能丢 set 记录。
- 计划修改不能回写破坏已完成历史。

## 5. 状态流转

- 本模块状态必须通过 Repository 写入 SQLite 或由 Domain 纯函数派生。
- 涉及训练执行的数据不得只存在组件局部状态。

## 6. 权限规则

第一版为本地默认小组，不做账号鉴权。模型和 Repository 需保留后续 owner/member/coach/guest 扩展空间。

## 7. 输入输出

- 输入：来自 UI 表单、SQLite Repository、seed 数据或其他 Domain 模块。
- 输出：领域对象、派生结果、Repository 持久化结果或 UI 可展示 ViewModel。

## 8. 异常处理

- 输入缺失时返回可解释状态，不让 UI 崩溃。
- SQLite 写入失败必须上抛或返回错误结果。
- 不允许静默丢弃训练现场输入。

## 9. 边界条件

- Sprint 4 已落地按动作轮换执行页；每个成员的每组记录独立保存到 `workout_sets`。
- 训练执行页使用大按钮 stepper 和 完成/跳过按钮，不做 Excel 表格。
- 动作替换已接入训练执行页，后续可继续补充替换原因和备注字段。

## 10. 训练中 UI 设计（2026-06-26 更新）

### 设计原则
- 遵循 impeccable skill 设计规范
- 品牌色 `#FF5A4D` 作为唯一强调色
- 无边框卡片，使用阴影层级
- 大量留白，清晰视觉层次

### 当前记录组件 (CurrentSetRecorder)
- 重量和次数使用大号 stepper 输入，背景 `surfaceMuted`
- 旧强度输入已从训练页移除
- 单一 CTA 按钮："完成本组"（品牌色）；休息中显示独立休息面板。
- 移除"跳过动作"按钮（用户可通过底部进度条切换动作）

### 已完成记录 (CompletedSetList)
- Chip 样式展示，每个记录独立一行
- 编号徽章（品牌色圆形）+ 成员名 + 重量/次数 chip
- 完成状态使用清晰按钮态区分
- 编辑和删除按钮内联显示

### 动作列表 (WorkoutProgressStrip)
- Dock 模式：进度条 + 动作 chip 标签
- Card 模式：垂直卡片列表，编号圆点 + 动作名
- 当前动作：品牌色边框 + "当前"徽章
- 已完成：绿色勾号

### 休息状态
- 休息时间独立：每个成员有自己的倒计时
- 休息面板展示倒计时、建议休息、已休息、下一组和下一位
- 其他成员休息状态：独立小卡片显示

## 11. 扩展点

- Repository 接口可增加远程同步实现。
- Domain 函数应保持可测试、可复用。
- seed 数据可替换为用户自定义计划或导入计划。

## 12. 修改风险

- 训练现场输入必须低摩擦，不能频繁要求键盘输入。
- 训练进行中退出 App 不能丢 set 记录。
- 计划修改不能回写破坏已完成历史。
