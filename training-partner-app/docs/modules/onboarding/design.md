# Onboarding 模块设计文档

更新时间：2026-06-30

## 设计目标

- 让新用户先回答少量训练相关问题，再进入今日训练。
- 推荐页必须真实影响当前计划，不做静态展示。
- 表单视觉参考“计划推荐”流程图，但使用练刻现有移动端组件和品牌体系。

## 流程结构

1. 完善训练信息：昵称、性别、年龄、身高、体重。
2. 训练目标与频率：增肌、增力、减脂保肌、新手入门、恢复训练；每周 2 / 3 / 4 / 5 天或不固定。
3. 训练基础：训练经验、器械条件、可选深蹲 / 卧推 / 硬拉 1RM。
4. 推荐计划：突出主推荐，展示目标、频率、经验、器械、结构和推荐理由，同时展示备选计划。

## 交互规则

- 非必填项可跳过。
- 1RM 缺失不阻塞推荐计划。
- 推荐计划点击“使用此计划”后复制为用户计划并设为当前计划。
- 备选计划可作为候选展示，后续可扩展为详情预览。
