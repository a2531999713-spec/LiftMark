# Recovery 模块设计文档

更新时间：2026-06-30

## 2026-06-30 动作筛选 UI 规则

- 用户入口命名为“动作筛选”，不再显示状态类旧称。
- “完整动作”保留 A/B/C；“精简辅助”隐藏 C；“只做主项”只保留 A；“今日休息”不创建训练 session。
- 该筛选只影响本次 session 快照，不修改原始计划模板。

## 1. 设计目标

让 `recovery` 模块拥有清晰边界，支撑多人训练 MVP，并为后续云同步、计划编辑器或商业版扩展保留空间。

## 2. 模块职责

- RecoveryLog 数据模型。
- 恢复评分 totalScore 和 recommendation。
- normal、remove_c、only_a、reduce_weight、deload、rest 建议。
- 动作筛选对动作优先级过滤的规则。

## 3. 非职责

- 不保存训练 set。
- 不直接改计划模板。
- 不替代用户的手动休息决定。

## 4. 核心业务规则

- 关节不适高分应优先休息或只做 A，不能硬推训练。
- 恢复评分输入可能缺失，今日训练应有默认模式。

## 5. 状态流转

- 本模块状态必须通过 Repository 写入 SQLite 或由 Domain 纯函数派生。
- 涉及训练执行的数据不得只存在组件局部状态。

## 6. 权限规则

第一版为本地默认小组，不做账号鉴权。模型和 Repository 需保留后续 owner/member/coach/guest 扩展空间。

## 7. 输入输出

- 输入：来自 UI 表单、SQLite Repository、seed 数据或其他 Domain 模块。
- 输出：领域对象、派生结果、Repository 持久化结果或 UI 可展示 ViewModel。

## 8. 异常处理

- 输入缺失时返回可解释状态，不让 UI 崩溃。
- SQLite 写入失败必须上抛或返回错误结果。
- 不允许静默丢弃训练现场输入。

## 9. 边界条件

- Sprint 3 已实现恢复评分和 A/B/C 过滤，但恢复日志持久化仍待后续 Sprint。
- 今日训练可手动选择动作筛选；后续接入 recovery_logs 时需保持同一过滤规则。

## 10. 扩展点

- Repository 接口可增加远程同步实现。
- Domain 函数应保持可测试、可复用。
- seed 数据可替换为用户自定义计划或导入计划。

## 11. 修改风险

- 关节不适高分应优先休息或只做 A，不能硬推训练。
- 恢复评分输入可能缺失，今日训练应有默认模式。
