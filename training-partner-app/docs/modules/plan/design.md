# Plan 模块设计文档

更新时间：2026-06-30

## 2026-06-30 主流计划库与推荐流程

- 系统方案目录改为主流力量训练计划库，不再在用户入口推荐旧“四练”兼容方案。
- 新增“经典四分化增肌计划”，按胸 + 三头、背 + 二头、肩、腿四天拆分，作为每周 4 天增肌用户的主流备选。
- 默认用户计划切换为“新手全身训练计划”；旧四练 seed 仅作为兼容历史数据的 legacy 模板保留，不进入可选系统方案和“我的计划”列表。
- 首次登录后进入训练信息完善流程，根据目标、频率、经验和器械条件推荐 3 个可执行方案。
- 推荐页主方案参考“计划推荐”视觉：深色推荐卡展示推荐理由、目标、频率、经验和器械条件，备选方案作为简洁列表。

## 2026-06-29 计划页趋势和工具收纳

- “本周执行和趋势”统一使用折线趋势图，不再使用柱状或胶囊柱表达最近 4 周执行次数。
- “计划工具”不再作为页面中部大卡片或工具网格暴露；创建、导入、导出和管理全部计划收进顶部更多操作和弱入口底部弹层。
- 当前计划主卡只保留去训练页等高频动作；导出当前计划属于低频动作，进入计划操作弹层。

## 1. 设计目标

计划模块要支撑“系统方案库”和“我的计划”分离：系统方案用于参考和复制，用户计划才是真正执行、编辑、导出和记录训练的对象。

## 2. 核心对象

| 对象 | 定位 | 是否可执行 | 是否可编辑 | 是否绑定训练记录 |
|---|---|---|---|---|
| SystemTrainingScheme | 系统只读方案目录 | 否 | 否 | 否 |
| UserTrainingPlan | 用户本地训练计划 | 是 | 是 | 是 |
| ActivePlan | 当前小组正在执行的计划状态 | 是 | 通过 UserTrainingPlan 修改 | 训练开始时生成快照 |
| TrainingRecord | 实际训练记录 | 不适用 | 不适用 | 绑定 UserTrainingPlan 或快照 |

## 3. 页面结构

计划页结构固定为：

```text
计划
- 当前计划仪表盘
- 本周执行和趋势
- 本周安排
- 我的计划摘要
- 管理全部计划
- 计划库入口
- 计划操作弹层：创建计划 / 导入计划 / 导出当前 / 管理全部
```

计划库使用 App 风格弹层承载系统方案。系统方案卡片显示目标、适合人群、频率、周期、训练日结构、简介、标签和可用状态。可用方案提供“预览”和“使用此方案”。开发中的方案收进“更多方案开发中”摘要卡，不在页面铺成长列表。

“使用此方案”必须使用 App 风格确认弹层：先说明系统会复制到“我的计划”，复制成功后再询问是否设为当前训练计划。不得把复制流程做成表单，也不得用系统 Alert 承载选择流程。

“管理全部计划”必须使用 App 风格底部弹层。每个用户计划展示计划名、来源、周期、频率和当前计划标签；可查看、设为当前、删除。删除必须二次确认。

## 4. 核心业务规则

- 系统方案不是用户计划。
- 系统方案不会直接出现在“我的计划”中。
- 用户点击“使用此方案”后，才复制生成新的用户计划。
- 复制出的用户计划 `source` 为 `system_copy`，并记录 `originSchemeId`。
- 当前计划只能指向用户计划；训练页读取的是 `groups.active_plan_id` 指向的用户计划。
- 训练页“切换计划”只列出“我的计划”，系统方案必须先复制。
- 训练页“切换计划”使用底部计划列表和二次确认卡，计划项展示来源、训练日数量或周期信息、当前计划标签。
- 训练记录不能直接绑定系统方案。
- 导出对象是用户计划，不导出系统方案、训练记录、成员 1RM、身体数据、激活码或设备信息。
- 导入计划必须重新生成本地 ID，不覆盖已有计划，导入后为可执行的用户计划。
- 导入计划内的动作按名称复用本机已有动作；缺失动作才写入 SQLite，自定义动作来源保留为 `custom`。
- 用户计划可以删除；系统方案不可删除，当前计划不可删除，最后一个用户计划不可删除。
- 删除用户计划不删除训练记录，历史记录仍按训练时快照和已保存 ID 展示。
- 计划详情入口第一版为只读摘要页，展示计划来源、周期、训练日和动作摘要，后续再在同一入口接入深度编辑。

## 5. 当前系统方案

- 新手全身训练计划：完整可用，默认推荐，适合刚开始训练或每周 2-3 天训练。
- Push Pull Legs 三分化计划：完整可用，推 / 拉 / 腿三天循环，适合每周 3 天训练。
- 经典四分化增肌计划：完整可用，胸 + 三头、背 + 二头、肩、腿四天拆分，适合每周 4 天稳定增肌。
- 上肢 / 下肢分化计划：完整可用，适合每周 4 天稳定训练。
- 5x5 基础力量计划：完整可用，围绕深蹲、卧推、硬拉和推举建立基础力量。
- 减脂保肌训练计划：完整可用，控制训练量并保留主要力量动作。
- 恢复训练计划：完整可用，适合疲劳较高或恢复期。
- 居家哑铃训练计划：完整可用，适合只有哑铃或自重条件的用户。

旧 Excel 四练兼容模板仍保留在 seed 与迁移中，目的仅是兼容已经存在的训练记录和旧默认计划 ID；新用户入口、系统方案列表和计划切换列表不展示该 legacy 方案。

## 6. 扩展点

第一阶段只做本地系统方案库。后续可以接远程目录：

```text
remote scheme catalog -> catalog.json -> scheme package url -> 下载 -> 使用此方案 -> 复制为 UserTrainingPlan
```

后台可扩展管理员上传、设置方案标题/目标/难度/标签、发布 catalog 和 App 拉取方案列表。

## 7. 经典三分化 PPL

方案结构：

- Day 1：推 Push，包括杠铃卧推、上斜哑铃卧推、推举、侧平举、绳索下压或臂屈伸。
- Day 2：拉 Pull，包括硬拉或罗马尼亚硬拉、引体向上或高位下拉、划船、面拉或反向飞鸟、哑铃弯举。
- Day 3：腿 Legs，包括深蹲、腿举或前蹲、罗马尼亚硬拉或腿弯举、保加利亚分腿蹲、提踵。

进阶规则使用双进阶：所有工作组达到目标次数上限且 完成率稳定 时下次加重；未达到下限维持重量；连续两次低于下限或 连续未完成时下次降重 5%-10%。
