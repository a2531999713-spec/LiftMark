# Plan 模块原则文档

更新时间：2026-06-12

## 1. 核心原则

- 训练方案是系统提供的模板或参考方案，不直接执行，不直接记录训练。
- 训练计划是用户真正使用的本地计划，可以编辑、设为当前、导出和绑定训练记录。
- 用户必须点击“使用此方案”或导入/创建后，才生成自己的计划。
- 训练记录只能绑定用户计划或训练时计划快照，不能直接绑定系统方案。

## 2. 数据原则

- 计划是数据，不是代码。
- 计划模板和个人参数分离。
- 计划和训练记录分离。
- 系统方案和用户计划分离。
- 本地 SQLite 优先。
- Domain 层不依赖 UI。
- 训练记录不能用 AsyncStorage 保存。

## 3. 命名原则

用户可见文案统一使用：

```text
系统方案
我的计划
当前计划
创建计划
导入计划
导出计划
使用此方案
复制为我的计划
设为当前计划
```

避免在新页面中使用“内置计划”“默认计划库”“系统计划”“模板计划”等混淆词。

## 4. 修改限制

- 不要把系统方案列表写成页面局部常量，方案目录应在 domain/data 层。
- 不要把训练日动作硬编码进 React 组件。
- 不要修改系统方案 seed 后自动覆盖用户已复制出的计划。
- 不要在未确认的情况下物理删除用户计划。
- 不要让导入计划覆盖已有计划。

## 5. 安全考虑

计划文件导出不得包含：

- 训练记录。
- 成员 1RM。
- 身体数据。
- 激活码。
- 设备信息。

危险操作例如删除计划、重置系统方案数据、清空测试数据必须二次确认。
