# Training Record 模块设计文档

更新时间：2026-06-14

## 本 Sprint 设计结论

- 记录页默认显示当前成员个人数据。
- 页面必须明确展示“当前成员：xxx”。
- 本周指标文案使用“我的本周训练量”“我的本周训练次数”“我的本周完成组数”。
- 小组汇总不能和个人数据混在一起。
- 如果小组汇总未实现，显示“开发中”说明，不复用个人数据假装小组统计。
- 2026-06-14 重新设计记录页基础版：页面从大表单/大卡片改为更紧凑的数据页，顶部仅保留一个日历入口，主体使用个人口径概览卡、2x2 小统计、基础趋势、可点击日历和紧凑记录列表。
- 设计稿已生成：`docs/ui/record-page-redesign.png`。

## 统计口径

- 当前版本的记录页个人统计只计算本机当前成员的训练记录。
- 单组训练量 = 重量 x 次数。
- 周训练量 = 本周完成组训练量逐组相加。
- 周训练次数 = 本周当前成员至少完成一组的 session 数。
- 周完成组数 = 本周当前成员 completed set 数。
- 估算 1RM 使用 Epley：重量 x (1 + 次数 / 30)，仅在重量 > 0 且次数 1-12 时计算。

## 日历与历史详情

- 周历和月视图日期可点击。
- 点击日期后显示当天当前成员训练记录。
- 当天无记录时显示友好空状态，并允许补录当天训练。
- 最近训练卡片以紧凑列表展示训练日期、训练标题、当前成员、动作数量、完成组数、总训练量、训练时长，以及查看/编辑入口。
- 点击训练卡片进入训练详情。

## 基础分析

- 分析能力第一版是基础规则建议，文案应说明依据完成组、RPE/RIR 和历史趋势生成。
- 趋势、PR 接近度、疲劳提示和建议基于本地历史记录计算。
- 间歇时间为空不影响训练量、PR 或估算 1RM。
