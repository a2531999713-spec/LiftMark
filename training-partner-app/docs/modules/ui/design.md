# UI 模块设计

更新时间：2026-06-30

## 2026-06-30 训练选择与低频入口规则

- 今日训练页的动作取舍入口命名为“动作筛选”，不得继续使用旧状态类命名作为用户主文案；它影响本次 session 的计划动作快照。
- 计划页主界面不铺开系统方案长列表；系统方案统一进入“计划库”弹层。
- 历史详情默认只读，编辑态和删除动作由更多菜单触发，避免普通查看记录时误操作。
- 普通二级页只保留系统返回，不叠加重复标题和大块顶部留白。

## 2026-06-29 记录 / 计划 / 同步入口 UI 规则

- 记录页支持个人 / 小组分段视角；小组视角必须展示本机 SQLite 推导的总览、成员贡献、折线趋势、最近小组训练和洞察，不得用开发中占位或假头像假数据。
- 记录页、训练分析页、计划执行趋势统一使用折线趋势图；不再使用柱状图、胶囊柱或椭圆柱表达训练量与趋势。
- 计划页低频工具收进“计划操作”底部弹层；页面中部不展示大块“计划工具”网格。
- 普通用户界面不展示“云同步 / 数据同步 / 待同步 / 同步状态 / 从云端恢复”等入口或模块；底层预留代码不作为用户主流程展示。
- 在线邀请、二维码加入、在线同练等未实现能力不作为首页或搭子页主入口展示；统一使用“该功能正在开发中，后续版本开放。”说明。

## 页面布局原则

- 首页是今日训练任务中心：今天练什么、今日重点、当前搭子、本周简要概览和开始训练入口。
- 计划页是计划管理中心：当前计划、阶段进度、本周安排、我的计划、系统方案、创建、复制、导入、导出。
- 记录页是数据分析中心：预估 1RM、PR、完成率、训练量、趋势、最近训练和分析建议。
- 我的页是账号与训练入口：训练档案、小组成员、偏好设置、账号设置和关于练刻；退出登录放在账号设置。
- 训练执行页使用沉浸式全屏，不显示底部 Tab。
