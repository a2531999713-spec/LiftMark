# UI 模块设计

更新时间：2026-06-14

## 2026-06-24 我的页视觉规则

- 我的页使用浅灰背景、白色分组卡、深海军蓝账号身份主卡、珊瑚红强调色、线性 Ionicons 和大圆角轻阴影。
- 顶部主卡不得展示云同步、会员权益、Pro 状态、待同步数量、会员到期时间或升级入口。
- 分组列表保持移动端单列，左右边距约 16px，列表项点击区域足够大，底部内容不能被 Tab 遮挡。
- 开发者诊断默认隐藏，通过连续点击版本号 7 次开启；普通用户不直接看到 SQLite、seed 或调试信息。

## 2026-06-24 登录 UI 规则

- 登录 / 注册页是极简手机号验证码表单，首屏直接展示品牌、手机号、验证码、获取验证码、登录 / 注册和协议勾选。
- 当前 UI 不展示继续浏览、预览模式、账号密码登录、账号密码注册、微信登录、Apple 登录、会员权益或小组协作卖点。
- 登录提示弹窗只保留登录动作，不提供关闭后浏览。
- 表单控件必须有 loading、disabled、错误提示和倒计时状态，错误文案必须是中文。

## 视觉方向

练刻 LiftMark 的界面应表现为成熟的力量训练工具：专业、干净、训练现场友好、数据清楚。整体以浅灰页面背景、白色卡片、深海军蓝训练场景和珊瑚红主行动形成层级。

## 品牌色

- Ink Navy：`#0D1B2A`
- Coral Red：`#FF5A4D`
- Light Gray：`#F2F4F7`
- White：`#FFFFFF`

所有页面通过 `src/theme/` 引用颜色、字号、间距、圆角和阴影，不在页面散落基础视觉参数。

## 页面布局原则

- 首页是今日训练任务中心：今天练什么、今日重点、当前搭子、本周简要概览和开始训练入口。
- 计划页是计划管理中心：当前计划、阶段进度、本周安排、我的计划、系统方案、创建、复制、导入、导出。
- 记录页是数据分析中心：预估 1RM、PR、完成率、训练量、趋势、最近训练和分析建议。
- 我的页是移动设置中心：成员资料、当前小组、默认单位、试用激活、数据管理、开发诊断和危险操作。
- 训练执行页使用沉浸式全屏，不显示底部 Tab。

## 资源设计

- 品牌图标和 wordmark 由 `src/assets/brand/index.ts` 的 `liftmarkBrandAssets` 导出。
- 训练氛围图由 `src/assets/images/index.ts` 的 `liftmarkImages` 导出。
- 页面不得直接引用原始参考图文件名。
- Android 原生图标和启动页资源已同步到 `android/app/src/main/res/`，本地 APK 构建不依赖重新 prebuild。

## 页面与弹层规则

- 除成员编辑、计划编辑、历史补录等真实输入流程外，普通功能页禁止做成表单。
- 设置页、计划页、记录页、探索页、训练页弹框必须使用 App 风格卡片、列表、标签、底部弹层或分组操作。
- 新增 Alert 只能用于危险确认、系统错误或极轻量输入错误，不得承载复杂选择流程。
- 未实现功能应收进说明卡、标签或二级入口；不要把显眼按钮做成点击后只提示“开发中”。
- `AppModalSheet` 用于复制方案确认、计划切换确认、导入后设为当前、导出内容复制等流程。
