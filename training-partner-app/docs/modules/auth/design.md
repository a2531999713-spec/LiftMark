# Auth 模块设计

更新时间：2026-06-29

## 1. 使用模式

| 模式 | 能做什么 | 限制 |
|---|---|---|
| `unauthenticated` | 只能看到登录 / 注册页 | 必须联网完成首次账号验证 |
| `offline_authenticated` | 有本地 session 时进入默认主界面和本机模式 | 不拉取云端完整资料，不做全量同步 |
| `logged_in_free` | 真实训练、离线保存、基础成员/小组/计划 | 超过免费额度进入 Pro 引导 |
| `logged_in_pro` | Pro 小组、高级历史、完整权益入口 | 支付和在线同练仍按功能进度开放 |
| `logged_in_lifetime` | 个人 Pro 权益长期有效 | 在线服务仍遵守合理使用和后续服务边界 |

## 2. 拦截体验

- 登录要求：未登录直接进入 `app/account/login.tsx`，主应用 Tab 不提供未登录预览。
- 登录要求弹窗只展示品牌、说明和“登录 / 注册”主按钮，不提供关闭后继续浏览。
- Pro 要求：标题“开通 Pro，解锁完整训练小组”，主按钮“查看 Pro 权益”，次按钮“暂不开通”。
- 未实现功能必须显示“该功能正在开发中，后续版本开放。”或同等明确提示。

## 2.1 登录页

- `app/account/login.tsx` 是短信验证码登录 / 注册一体页，不再提供未登录预览或第三方登录入口。
- 登录支持手机号 + 短信验证码：手机号 -> 获取验证码 -> 输入验证码 -> 登录 / 注册。
- 新手机号由后端验证码登录接口自动创建账号；移动端不再展示密码注册表单。
- 验证码同时服务登录、自动创建账号和后续绑定类敏感流程。
- 必须勾选“我已阅读并同意《用户协议》和《隐私政策》”后才能发送验证码或登录。
- 验证码发送必须有手机号格式校验、协议校验、loading、60 秒倒计时和中文错误提示。
- 登录页不展示继续浏览、预览模式、第三方登录、会员权益、Pro 说明或小组协作卖点。

## 3. 本地优先

- 登录是正式数据归属和云同步前提，不是训练执行联网前提。
- 训练记录继续先写 SQLite。
- 退出登录不清空本机成员、计划或训练记录。
- 离线进入本机模式时不清空 session，不清空 SQLite，不请求云端完整数据。
