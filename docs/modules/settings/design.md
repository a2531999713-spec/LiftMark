# Settings 模块设计文档

更新时间：2026-06-14

## 2026-06-14 设置页重设计

- 设置页从大块表单卡片改为移动端设置中心：顶部品牌/状态卡 + 分组列表项。
- 设计稿已生成：`docs/ui/settings-page-redesign.png`。
- 分组包括：通用设置、训练设置、试用与激活、数据管理、计划管理、开发与诊断、危险操作。
- 未实现能力保留“开发中”标签或弹窗，不允许无响应。
- 设置页可通过“本地小组规则”入口说明当前本地小组与未来云同步区别，但不在主页面堆长段落。

## 1. 设计目标

设置页作为本地优先 MVP 的安全控制面板：能让用户看到当前 App 和数据状态，并访问导出、导入、重置等高风险能力。

## 2. 核心业务规则

- App 品牌显示为“练刻 LiftMark”。
- 第一阶段导出以 JSON 字符串预览为基础，文件保存/分享后续接入。
- `.liftmark` 计划文件只包含计划模板、动作库、替代动作和进阶规则，不包含成员 1RM 或训练记录。
- 导入计划必须生成新的本地 ID，避免覆盖既有计划。
- 重置默认计划只补齐 seed 缺失数据，不删除训练记录。
- 清空测试数据暂不真正执行删除，等待软删除/清空策略确认。

## 3. 输入输出

- 输入：SQLite Repository、导出服务、计划文件服务。
- 输出：设置页状态、导出 JSON 预览、确认弹窗、开发中提示。

## 4. 边界条件

- SQLite 初始化失败时显示中文错误。
- 未实现功能必须弹出“开发中”。
- 危险操作必须二次确认。
