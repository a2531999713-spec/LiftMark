# Training 模块设计文档

更新时间：2026-06-14

## 本 Sprint 设计结论

- 训练执行页从“手动停留当前组”改为“完成本组后自动推进”。
- 点击“完成本组”后先保存当前组数据；当前动作还有下一组时进入休息计时；休息结束后进入下一组；当前动作完成后自动进入下一个动作；最后一个动作完成后显示“完成训练并查看总结”。
- 保留手动按钮：跳过休息、下一组、下一个动作、完成训练并查看总结。
- 如果计划动作没有设置休息时间，完成本组后直接进入下一组，不强制休息。
- 当前动作卡片、当前组输入卡片、休息倒计时卡片、已完成组列表和底部主操作按钮共同构成训练现场页面。

## 输入规则

- 重量支持点击数字直接输入，允许小数，例如 97.5。
- 次数支持点击数字直接输入，只允许整数。
- RPE 不再使用键盘输入，只能点选 `6 / 7 / 8 / 9 / 10 / 清空`。
- RIR 不再使用键盘输入，只能点选 `0 / 1 / 2 / 3 / 4 / 5 / 清空`。
- RPE/RIR 选中态必须明显；清空代表本组不记录该字段。
- 非法输入必须提示用户，不允许崩溃或写入脏数据。
- 加减号保留，用于现场快速微调。

## 已完成组编辑规则

- 当前动作已完成组必须可见，例如“第 1 组：97.5kg x 5，RPE 8”。
- 每一组支持编辑和删除。
- 编辑已完成组时可修改重量、次数、RPE、RIR 和备注。
- 删除已完成组需要二次确认。
- “撤销上一组”需要二次确认，当前实现是把当前动作最新完成的组恢复为未完成，不新增重复记录。

## 数据口径

- 单组训练量 = 重量 x 次数。
- 如果每组已经单独记录，训练日总量只对完成组逐组相加，不再额外乘组数。
- 估算 1RM 使用 Epley：重量 x (1 + 次数 / 30)，只在重量 > 0 且次数 1-12 时计算。
- 历史记录必须保留动作、计划、训练日和组数据快照；后续编辑计划或动作不应破坏旧记录展示。

## 非目标

- 本 Sprint 不做动作 GIF/视频。
- 不做账号登录、云同步、饮食模块、激活码或品牌迁移。
- 不把训练记录写入 AsyncStorage。
