# 系统方案复制为用户计划流程

更新时间：2026-06-12

## 1. 目标

系统方案是只读模板，不能直接执行训练。用户必须点击“使用此方案”后，App 才复制生成一份本地用户计划。

## 2. 主流程

```text
打开计划页
-> 查看系统方案
-> 点击“查看详情”
-> 点击“使用此方案”
-> 输入或确认计划名称
-> 点击“复制为我的计划”或“复制并设为当前”
-> Repository 复制系统模板数据
-> 写入新的 UserTrainingPlan
-> 可选更新 groups.active_plan_id
-> 训练页读取新的用户计划
```

## 3. 数据规则

- 新计划 `source = system_copy`。
- 新计划记录 `originSchemeId`。
- phases、days、plan exercises 都生成新的本地 ID。
- 系统方案不会进入“我的计划”。
- 训练记录不会绑定系统方案。

## 4. 异常流程

- 系统方案开发中：显示“开发中”。
- 系统模板不存在：显示中文错误提示。
- SQLite 写入失败：复制失败并提示，不创建半成品计划。

## 5. 验收点

- “我的计划”只显示用户计划。
- 点击完整可用系统方案后能生成用户计划。
- 用户可以把生成计划设为当前计划。
- 今日训练读取当前用户计划。
