# Hide Export Preview to Developer Mode Only

## Problem

The "最近导出预览" (Recent Export Preview) card is visible to all users in the Settings page. This is a development/debug feature that should only be visible in development mode.

## Solution

Move the export preview card into the existing "开发与诊断" (Dev & Diagnostics) section, and wrap it with a `__DEV__` check so it only renders in development builds.

## File to Modify

`app/(tabs)/settings.tsx`

## Changes

### 1. Remove the standalone preview card at bottom (lines 355-364)

Delete:
```tsx
{exportPreview ? (
  <AppCard style={styles.previewCard}>
    <AppText variant="bodySmall" weight="900">
      最近导出预览
    </AppText>
    <AppText tone="muted" variant="caption">
      {exportPreview.slice(0, 900)}
    </AppText>
  </AppCard>
) : null}
```

### 2. Add preview inside the "开发与诊断" group (after line 346)

Insert after the existing dev items:
```tsx
{__DEV__ && exportPreview ? (
  <AppCard style={styles.previewCard}>
    <AppText variant="bodySmall" weight="900">
      最近导出预览
    </AppText>
    <AppText tone="muted" variant="caption">
      {exportPreview.slice(0, 900)}
    </AppText>
  </AppCard>
) : null}
```

## Verification

1. In development mode (`npx expo start`), export data and verify preview appears in "开发与诊断" section
2. In production build (`npx expo run:android --release`), verify preview is hidden
3. Ensure the "开发与诊断" section still shows other diagnostic items correctly
